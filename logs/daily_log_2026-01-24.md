### 프로젝트 초기 구조 설정 및 핵심 문서 추가

- **Type**: 신규 기능 (Feature)
- **Details**:
    - KAIROS 프로젝트의 초기 디렉토리 구조를 설정하고, 향후 개발을 위한 포괄적인 기반을 마련했습니다.
    - **`.gitignore` 파일 추가**:
        - 빌드 결과물(`*.class`, `**/build/`, `**/target/`), 로그 파일, IDE 설정(`/.idea/`, `/.vscode/`, `/.agent/`) 등 불필요한 파일들이 Git 추적에서 제외되도록 설정했습니다.
        - `application-secret.yml`, `.env` 파일 등 민감 정보가 포함될 수 있는 파일들을 명시적으로 제외하여 보안을 강화했습니다.
    - **핵심 설계 및 사양 문서 대량 추가**:
        - `docs/` 디렉토리 내에 프로젝트의 전체적인 방향과 아키텍처, 개발 계획을 정의하는 핵심 문서들을 추가했습니다.
        - **`Master_Plan.md` & `PROJECT-Specification.md`**: Java 21 가상 스레드 기반 아키텍처, 7-Agent 시스템, TDD 원칙 등 프로젝트의 전체 개발 로드맵과 기술 사양을 정의했습니다.
        - **`AI_Agent_List.md`**: 7인의 AI 에이전트(Sentinel, Axiom, Vector 등) 각각의 역할, 페르소나, 모델(Gemini Flash/Pro), 시스템 프롬프트를 상세하게 명세했습니다.
        - **`EXT_API_Specification.md` & `INT_API_Specification.md`**: 키움증권, Naver, Gemini 등 외부 API 연동 규격과 프론트엔드와 통신할 내부 API의 엔드포인트 및 데이터 모델을 상세하게 정의했습니다.
        - **`Workflow_and_UI_Spec.md`**: 시간대별 시스템 워크플로우, AI 의사결정 로직, 트레이딩 알고리즘, UI/UX 상세 명세를 정의하여 개발의 구체적인 가이드라인을 제공했습니다.
        - **`Benchmark_Report.md`**: `prism-insight` 오픈소스 분석을 통해 KAIROS에 적용할 기술적 벤치마크 및 고도화 전략을 수립했습니다.
    - **외부 API 문서 통합**:
        - 키움증권 REST API의 상세 명세가 담긴 `키움 REST API 문서.pdf` 및 `.xlsx` 파일을 `docs/`에 추가하여 개발 편의성을 확보했습니다.
    - **LICENSE 파일 재배치**:
        - 최상위 디렉토리에 있던 `LICENSE` 파일을 `docs/` 디렉토리로 이동하여 문서 구조를 일원화했습니다.
